# 一个优秀的资金管理系统

首先将交易资金分成十份，其中一份用来作为保证金放在合约账户里，另外九份放在其他账户下。然后每次开仓都使用十倍杠杆，而实际杠杆为1，每次止损在总交易资金的1%-2%。如果亏损掉所有合约账户里的钱，则再把交易资金重新分配，把其中1/10放入合约账户，同理如果赚到10%，也进行重新分配，保证合约账户里的资金占总交易资金的10%。这套资金管理系统可以达到：**指数级增加资金增长的速度，指数级减少资金减少的速度。**

## 优点

- 分仓的最大优势在于，在极端黑天鹅行情下，最大损失金额是10%而不是100%。

- 连续亏损下的韧性

  | 亏损后重新分配次数 | 剩余资金 |
  | ------------------ | -------- |
  | 1                  | 90%      |
  | 2                  | 81%      |
  | 3                  | 73%      |
  | 4                  | 66%      |
  | 5                  | 59%      |
  | 6                  | 53%      |
  | 7                  | 48%      |
  | 8                  | 43%      |
  | 9                  | 39%      |
  | 10                 | 35%      |

  在连续损失全部合约账户10次的条件下，还剩大约三分之一的交易资金，几乎是一种永不爆仓的资金管理系统。在两年使用这套系统后，我最多也只走到了第二步，也就是交易资金回撤20%不到，越往后概率越低，我大胆估计，一个严格按照该资金管理系统执行的正常人，5以及后面触发的概率几乎为0可以忽略不计。

- 复利增长，复利的伟大，无需多言。

## 缺点

目前来看这套资金管理系统不存在明显缺点，唯一的问题就是资金利用效率过低，剩余9份资金放在其他账户吃利息，未来可以考虑放在其他地方，防止交易所暴雷跑路。
